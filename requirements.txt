# ML / RL
# torch's default Linux wheel (plain PyPI, no local version suffix) bundles
# CUDA/cuDNN/cuBLAS — multiple GB of NVIDIA shared libraries — even though
# nothing here touches a GPU (RL inference is CPU-only, see
# docs/architecture/adr/0001-rl-inference-only-in-production.md). The
# CPU-only build lives on PyTorch's own wheel index under a "+cpu" local
# version, hence --extra-index-url below plus a platform split: Linux (CI
# runners, the Docker image) gets the +cpu build; macOS never had a CUDA
# build to opt out of, so it keeps the plain PyPI version it already used.
# --extra-index-url is file-scoped, so it also applies when this file is
# pulled in via `-r requirements.txt` (requirements-dev.txt) — same pin
# everywhere, nothing depends on machine-local pip config.
--extra-index-url https://download.pytorch.org/whl/cpu
torch==2.13.0+cpu ; sys_platform == "linux"
torch==2.13.0 ; sys_platform != "linux"
